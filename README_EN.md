## ğŸ¹ How to Use?

### Prerequisites

- **Node.js**: >= 16.0.0
- **Package Manager**: pnpm (Recommended) / npm / yarn

### Clone the Repository

```bash
git clone https://github.com/nicejade/gpt-wordbook.git
cd gpt-wordbook
```

### Install Dependencies

```bash
# Using pnpm (Recommended)
pnpm install

# Or using yarn
yarn install

# Or using npm
npm install
```

### Start Development Server

```bash
# Using pnpm
pnpm start

# Or using npm
npm run start

# Or using yarn
yarn start
```

The development server will start at `http://localhost:6969`.

### Build Production Version

```bash
# Using pnpm
pnpm build
```

The static files will be output to the `dist/` directory after the build is complete.

### Preview Production Version Locally

```bash
# Using pnpm
pnpm preview
```

## ğŸš€ How to Deploy?

Since this project is a pure static website built with [Astro](https://astro.build/), you can easily deploy it to various platforms.

### Recommended Deployment Platforms

#### 1. **Cloudflare Pages** (Recommended)

- Free with unlimited traffic.
- Global edge network.
- Excellent performance.
- Continuous integration support.

#### 2. **Vercel**

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/nicejade/gpt-wordbook)

- Zero-configuration deployment.
- Automatic HTTPS.
- Global CDN acceleration.
- Continuous integration support.

#### 3. **GitHub Pages**

```bash
# Modify 'site' and 'base' configurations in 'astro.config.mjs'
# Then build and deploy
pnpm build
```

#### 4. **Netlify**

- Drag-and-drop deployment.
- Automatic builds.
- Extra features like form handling.

### Deployment Steps (e.g., Cloudflare Pages)

1. Fork this repository to your GitHub account.
2. In the [Cloudflare Dashboard](https://dash.cloudflare.com/), click **Workers & Pages** -> **Create application** -> **Pages** -> **Connect to Git**.
3. Select the `gpt-wordbook` repository and click **Begin setup**.
4. Configure build settings:
   - **Framework preset**: `Astro`
   - **Build command**: `pnpm build`
   - **Output directory**: `dist`
5. Click **Save and Deploy**.
6. After deployment, you will receive a `*.pages.dev` subdomain.
